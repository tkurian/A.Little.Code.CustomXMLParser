CUSTOM XML PARSER

This is a neat application that will take in XML files, parse them and create a way for the user to view all feeds which are of interest to them in one place!

High-Level Description

This application consists of 6 different packages. These packages are as follows:

1. management

The management package consists of 1 file. This file is as follows:

UIManager.java

This file will determine which OS you are running and launch a either a Windows GUI or a linux terminal in order to run the aplication.

2. linux

The linux package consists of 1 file. This file is as follows:

LinuxConsole.java

This file manages the user interface for the lin.x version.

3. windows

The windows package consists of 8 files. These file is as follows:

WindowsGUI.java

This file manages the graphical user interface for the Custo.XMLParser.

FeedListListener.java

This file handles the mouse events for the Feed list from the .indowsGUI.

ItemHtmlDisplay.java

This file creates a display string for displaying a RSS item to the screen. It also creates a CSS style that can be added to an element being added to a J.ditorPane.

ItemListListener.java

This file is the listener for the item list

RssFeedListCellRenderer.java

This file is used for displaying a feed list on .he screen.

RssFeedListModel.java

This file handles linking data between the RSS feed and the f.edManager.

RssItemListCellRenderer.java

This file is used for displaying the RSS items on.the JList.

RssItemListModel.java

This class links channel data to a JList display.

4. utilities

FileIO.java

This file will handle reading in the RSS feed/XML file. It will return back the contents of the xml file within a string.

RssFeed.java

This file handles loading the XML file into a string and setting the feed name, feed url and icon url. It also consists of getter methods in order for easy retrieval of the feed name, feed url and icon url.

RssFeedManager.java

This file handles the config file, adding feeds, removing feeds. The config file enables us to pre-load a few XMLs into our XMLParser. This means upon loading, some feeds will pre-exist (without the user having to add them). Hence, this file consists of methods to update the config file.

XMLParser.java

This file manages parsing the XML file/RSS feed. It will then populate the database. This requires parsing channels, images, items and attributes.

XMLValidation.java

This file consists of some very basic validation tests. We test to see if the version is 2.0 and we check to ensure that the RSS pattern is correct (ie. no missing tags!)

5. database

The database was designed to hold a list of channels. Each channel would consist of items and images. To facilitate this design, the database package had to consist of 4 different files. These files include the following:

DBase.java

This file is for handling the databaseList. The databaseList consists of channels. Hence it contains methods to add channels, remove channels, count the number of channels, get a specific channel, get the full list of channels or get a channel based on it's attributes (attribute name & value).

Channel.java

This file is for handling the channelList. The channel list consists of both images and items.

Item.java

This file is for creating Item objects that will be stored within the channelList.

Image.java

This file is for handling any images that need to be added to the channelList.

6. test

The testpackage consists of 1 file. This file is as follows:

ConsoleTestHarness.java

This file was used in order to test the methods being created, in order to ensure that they were functioning as intended.
